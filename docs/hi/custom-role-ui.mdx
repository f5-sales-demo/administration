यह गाइड F5 Distributed Cloud Console UI का उपयोग करके एक कस्टम RBAC भूमिका बनाने की प्रक्रिया बताती है। यह भूमिका HTTP Load Balancers द्वारा उपयोग किए जाने वाले SSL/TLS प्रमाणपत्रों और प्रमाणपत्र श्रृंखलाओं के प्रबंधन की पहुँच प्रदान करती है। प्रोग्रामेटिक (API) दृष्टिकोण के लिए, [API गाइड](./custom-role-api) देखें।

## F5 XC RBAC कैसे काम करता है

F5 XC एक तीन-स्तरीय अनुमति मॉडल का उपयोग करता है:

```
api_group_element  (read-only, system-defined)
       │
       ▼
   api_group       (read-only, system-defined)
       │
       ▼
     role           (full CRUD — this is what you create)
```

1. **`api_group_element`** — एक regex पथ और HTTP विधियों को परिभाषित करता है
   (उदा., `POST /api/config/.*/certificates`)। सिस्टम-प्रबंधित; आप इन्हें
   बना, संशोधित या हटा नहीं सकते।

2. **`api_group`** — `api_group_element` संदर्भों का एक नामित संग्रह,
   कार्यात्मक क्षेत्र के अनुसार व्यवस्थित। सिस्टम-प्रबंधित; आप इन्हें बना, संशोधित या
   हटा नहीं सकते। समूह `ves-io-proxy-read` या
   `f5xc-waap-standard-admin` जैसे नामकरण परंपराओं का उपयोग करते हैं।

3. **`role`** — एक array में `api_group` नामों को संदर्भित करता है। **यह एकमात्र
   स्तर है जो आप बनाते हैं।** विशिष्ट `api_groups` को किसी भूमिका से जोड़ने के लिए
   कस्टम भूमिका endpoints का उपयोग करें।

proxy read/write समूहों का चयन करके, `example-cert-admin` भूमिका व्यापक proxy संसाधन पहुँच के भाग के रूप में SSL/TLS प्रमाणपत्र प्रबंधन प्रदान करती है।

## दायरा और सीमाएँ

proxy read/write समूहों का चयन करने से असाइन किए गए namespace में **सभी proxy संसाधनों** तक पहुँच मिलती है — HTTP Load Balancers, origins, routes और प्रमाणपत्र सभी शामिल हैं। भूमिका को केवल प्रमाणपत्रों तक सीमित करने का कोई तरीका नहीं है।

F5 XC उपयोगकर्ता-निर्मित `api_group` या `api_group_element`
ऑब्जेक्ट्स का समर्थन नहीं करता। प्लेटफ़ॉर्म इन्हें read-only के रूप में पूर्व-परिभाषित करता है, और भूमिकाएँ केवल मौजूदा समूह नामों को संदर्भित कर सकती हैं। कोई भी सिस्टम-परिभाषित समूह विशेष रूप से प्रमाणपत्र संचालन तक सीमित नहीं है।

**शमन — namespace पृथक्करण:** प्रमाणपत्र संसाधनों को एक
समर्पित namespace में रखें और `example-cert-admin` भूमिका को उस namespace तक सीमित करें। यह
उपयोगकर्ता को अन्य namespaces में proxy संसाधनों तक पहुँचने से रोकता है जबकि
पृथक namespace के भीतर पूर्ण प्रमाणपत्र प्रबंधन प्रदान करता है।

## पूर्वापेक्षाएँ

- F5 Distributed Cloud Console तक व्यवस्थापक पहुँच

## चरण 1: भूमिका प्रबंधन पर जाएँ

1. F5 Distributed Cloud Console में लॉग इन करें
2. होम पेज पर **Administration** टाइल पर क्लिक करें
3. बाएँ नेविगेशन मेनू में, **IAM** &gt; **Roles** चुनें
4. **+ Add Role** पर क्लिक करें

## चरण 2: भूमिका का नाम दें

**Role Name** फ़ील्ड में `example-cert-admin` दर्ज करें।

भूमिका नामकरण नियम:

- केवल लोअरकेस अक्षर, संख्याएँ और हाइफ़न
- दो लोअरकेस अक्षरों से शुरू होना चाहिए
- अधिकतम 64 वर्ण

## चरण 3: Proxy API समूह जोड़ें

प्रमाणपत्र अनुमतियाँ proxy कार्यात्मक क्षेत्र का हिस्सा हैं। प्रमाणपत्र प्रबंधन पहुँच प्रदान करने के लिए निम्नलिखित API समूह जोड़ें:

1. **+ Allowed API Groups** पर क्लिक करें
2. खोज/फ़िल्टर फ़ील्ड में, `ves-io-proxy` टाइप करें
3. निम्नलिखित API समूहों का चयन करें:

| API समूह | प्रदान करता है |
| --------- | ------ |
| `ves-io-proxy-read` | प्रमाणपत्रों सहित proxy संसाधनों तक पठन पहुँच |
| `ves-io-proxy-write` | प्रमाणपत्रों सहित proxy संसाधनों तक लेखन पहुँच |

4. चयनित API समूहों की पुष्टि करने के लिए **Save** पर क्लिक करें

:::note
ये समूह सभी proxy-संबंधित संसाधनों (HTTP Load Balancers, origins,
routes, प्रमाणपत्र, प्रमाणपत्र श्रृंखलाएँ, आदि) को कवर करते हैं, केवल प्रमाणपत्रों को नहीं। F5 XC
अनुमतियों को व्यक्तिगत संसाधन प्रकार के अनुसार नहीं, बल्कि कार्यात्मक क्षेत्र के अनुसार व्यवस्थित करता है।
:::

## चरण 4: भूमिका सहेजें

सारांश में चयनित API समूहों की समीक्षा करें, फिर भूमिका बनाने के लिए **Save** पर क्लिक करें।

## चरण 5: भूमिका सत्यापित करें

1. **IAM** &gt; **Roles** पर जाएँ
2. भूमिका सूची में **example-cert-admin** पर क्लिक करें
3. पुष्टि करें कि दोनों API समूह सूचीबद्ध हैं

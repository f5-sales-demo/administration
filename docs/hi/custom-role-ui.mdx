यह गाइड F5 Distributed Cloud Console UI का उपयोग करके एक कस्टम RBAC भूमिका बनाने की प्रक्रिया बताती है। यह भूमिका HTTP Load Balancers द्वारा उपयोग किए जाने वाले SSL/TLS प्रमाणपत्रों और प्रमाणपत्र श्रृंखलाओं को प्रबंधित करने की पहुँच प्रदान करती है। प्रोग्रामेटिक (API) दृष्टिकोण के लिए, [API गाइड](./custom-role-api) देखें।

## F5 XC RBAC कैसे काम करता है

F5 XC एक तीन-स्तरीय अनुमति मॉडल का उपयोग करता है:

```
api_group_element  (read-only, system-defined)
       │
       ▼
   api_group       (read-only, system-defined)
       │
       ▼
     role           (full CRUD — this is what you create)
```

1. **`api_group_element`** — एक regex पथ और HTTP मेथड्स को परिभाषित करता है
   (जैसे, `POST /api/config/.*/certificates`)। सिस्टम-प्रबंधित; आप इन्हें
   बना, संशोधित या हटा नहीं सकते।

2. **`api_group`** — `api_group_element` संदर्भों का एक नामित संग्रह,
   कार्यात्मक क्षेत्र के अनुसार व्यवस्थित। सिस्टम-प्रबंधित; आप इन्हें बना,
   संशोधित या हटा नहीं सकते। समूह `ves-io-proxy-read` या
   `f5xc-waap-standard-admin` जैसी नामकरण परंपराओं का उपयोग करते हैं।

3. **`role`** — एक ऐरे में `api_group` नामों को संदर्भित करता है। **यह एकमात्र
   स्तर है जो आप बनाते हैं।** विशिष्ट `api_groups` को एक भूमिका से जोड़ने के
   लिए कस्टम भूमिका एंडपॉइंट्स का उपयोग करें।

प्रॉक्सी read/write समूहों का चयन करके, `example-cert-admin` भूमिका व्यापक प्रॉक्सी संसाधन पहुँच के हिस्से के रूप में SSL/TLS प्रमाणपत्र प्रबंधन प्रदान करती है।

## दायरा और सीमाएँ

प्रॉक्सी read/write समूहों का चयन करने से असाइन किए गए नेमस्पेस में **सभी प्रॉक्सी संसाधनों** तक पहुँच प्राप्त होती है — HTTP Load Balancers, origins, routes, और प्रमाणपत्र सभी शामिल हैं। भूमिका को केवल प्रमाणपत्रों तक सीमित करने का कोई तरीका नहीं है।

F5 XC उपयोगकर्ता-निर्मित `api_group` या `api_group_element` ऑब्जेक्ट्स का समर्थन नहीं करता। प्लेटफ़ॉर्म इन्हें केवल-पढ़ने योग्य के रूप में पूर्व-परिभाषित करता है, और भूमिकाएँ केवल मौजूदा समूह नामों को संदर्भित कर सकती हैं। कोई भी सिस्टम-परिभाषित समूह विशेष रूप से प्रमाणपत्र संचालन तक सीमित नहीं है।

**शमन — नेमस्पेस पृथक्करण:** प्रमाणपत्र संसाधनों को एक समर्पित नेमस्पेस में रखें और `example-cert-admin` भूमिका को उस नेमस्पेस तक सीमित करें। यह उपयोगकर्ता को अन्य नेमस्पेस में प्रॉक्सी संसाधनों तक पहुँचने से रोकता है जबकि पृथक नेमस्पेस के भीतर पूर्ण प्रमाणपत्र प्रबंधन प्रदान करता है।

## पूर्वापेक्षाएँ

- F5 Distributed Cloud Console तक व्यवस्थापक पहुँच

## चरण 1: भूमिका प्रबंधन पर जाएँ

1. F5 Distributed Cloud Console में लॉग इन करें
2. होम पेज पर **Administration** टाइल पर क्लिक करें
3. बाएँ नेविगेशन मेनू में, **IAM** &gt; **Roles** चुनें
4. **+ Add Role** पर क्लिक करें

## चरण 2: भूमिका का नाम दें

**Role Name** फ़ील्ड में `example-cert-admin` दर्ज करें।

भूमिका नामकरण नियम:

- केवल लोअरकेस अक्षर, संख्याएँ और हाइफ़न
- दो लोअरकेस अक्षरों से शुरू होना चाहिए
- अधिकतम 64 वर्ण

## चरण 3: प्रॉक्सी API समूह जोड़ें

प्रमाणपत्र अनुमतियाँ प्रॉक्सी कार्यात्मक क्षेत्र का हिस्सा हैं। प्रमाणपत्र प्रबंधन पहुँच प्रदान करने के लिए निम्नलिखित API समूह जोड़ें:

1. **+ Allowed API Groups** पर क्लिक करें
2. खोज/फ़िल्टर फ़ील्ड में, `ves-io-proxy` टाइप करें
3. निम्नलिखित API समूह चुनें:

| API समूह | प्रदान करता है |
| --------- | ------ |
| `ves-io-proxy-read` | प्रमाणपत्रों सहित प्रॉक्सी संसाधनों तक पढ़ने की पहुँच |
| `ves-io-proxy-write` | प्रमाणपत्रों सहित प्रॉक्सी संसाधनों तक लिखने की पहुँच |

4. चयनित API समूहों की पुष्टि करने के लिए **Save** पर क्लिक करें

:::note
ये समूह सभी प्रॉक्सी-संबंधित संसाधनों (HTTP Load Balancers, origins,
routes, प्रमाणपत्र, प्रमाणपत्र श्रृंखलाएँ, आदि) को कवर करते हैं, केवल प्रमाणपत्रों को नहीं। F5 XC
अनुमतियों को कार्यात्मक क्षेत्र के अनुसार व्यवस्थित करता है, व्यक्तिगत संसाधन प्रकार के अनुसार नहीं।
:::

## चरण 4: भूमिका सहेजें

सारांश में चयनित API समूहों की समीक्षा करें, फिर भूमिका बनाने के लिए **Save** पर क्लिक करें।

## चरण 5: भूमिका सत्यापित करें

1. **IAM** &gt; **Roles** पर जाएँ
2. भूमिका सूची में **example-cert-admin** पर क्लिक करें
3. पुष्टि करें कि दोनों API समूह सूचीबद्ध हैं

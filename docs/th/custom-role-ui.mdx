คู่มือนี้อธิบายขั้นตอนการสร้างบทบาท RBAC ที่กำหนดเองโดยใช้ UI ของ F5 Distributed
Cloud Console บทบาทนี้ให้สิทธิ์ในการจัดการใบรับรอง SSL/TLS และ
สายใบรับรอง (certificate chains) ที่ใช้โดย HTTP Load Balancers สำหรับแนวทางแบบโปรแกรม (API)
โปรดดู [คู่มือ API](./custom-role-api)

## หลักการทำงานของ RBAC ใน F5 XC

F5 XC ใช้โมเดลสิทธิ์แบบสามระดับ:

```
api_group_element  (read-only, system-defined)
       │
       ▼
   api_group       (read-only, system-defined)
       │
       ▼
     role           (full CRUD — this is what you create)
```

1. **`api_group_element`** — กำหนดเส้นทาง regex และ HTTP methods
   (เช่น `POST /api/config/.*/certificates`) ระบบจัดการให้; คุณไม่สามารถ
   สร้าง แก้ไข หรือลบได้

2. **`api_group`** — คอลเลกชันที่ตั้งชื่อแล้วของการอ้างอิง `api_group_element`
   จัดระเบียบตามพื้นที่ฟังก์ชัน ระบบจัดการให้; คุณไม่สามารถสร้าง แก้ไข หรือ
   ลบได้ กลุ่มใช้แบบแผนการตั้งชื่อเช่น `ves-io-proxy-read` หรือ
   `f5xc-waap-standard-admin`

3. **`role`** — อ้างอิงชื่อ `api_group` ในอาร์เรย์ **นี่เป็นระดับเดียวที่คุณสร้าง** ใช้เอนด์พอยต์ของบทบาทที่กำหนดเองเพื่อแนบ
   `api_groups` เฉพาะเข้ากับบทบาท

เมื่อเลือกกลุ่ม proxy read/write บทบาท `example-cert-admin` จะให้สิทธิ์การจัดการใบรับรอง SSL/TLS
เป็นส่วนหนึ่งของการเข้าถึงทรัพยากร proxy ในวงกว้าง

## ขอบเขตและข้อจำกัด

การเลือกกลุ่ม proxy read/write จะให้สิทธิ์เข้าถึง **ทรัพยากร proxy ทั้งหมด**
ใน namespace ที่กำหนด — HTTP Load Balancers, origins, routes และ
ใบรับรอง ทั้งหมด ไม่มีวิธีจำกัดบทบาทให้เฉพาะใบรับรองเท่านั้น

F5 XC ไม่สนับสนุนให้ผู้ใช้สร้างอ็อบเจกต์ `api_group` หรือ `api_group_element`
แพลตฟอร์มกำหนดสิ่งเหล่านี้ล่วงหน้าเป็นแบบอ่านอย่างเดียว และบทบาทสามารถ
อ้างอิงชื่อกลุ่มที่มีอยู่แล้วเท่านั้น ไม่มีกลุ่มที่ระบบกำหนดไว้ที่ครอบคลุมเฉพาะ
การดำเนินการเกี่ยวกับใบรับรองเพียงอย่างเดียว

**การบรรเทา — การแยก namespace:** วางทรัพยากรใบรับรองใน
namespace เฉพาะ และกำหนดขอบเขตบทบาท `example-cert-admin` ให้กับ namespace นั้น วิธีนี้
ป้องกันไม่ให้ผู้ใช้เข้าถึงทรัพยากร proxy ใน namespace อื่น ขณะที่
ยังคงให้สิทธิ์การจัดการใบรับรองอย่างเต็มที่ภายใน namespace ที่แยกไว้

## ข้อกำหนดเบื้องต้น

- สิทธิ์ Admin ในการเข้าถึง F5 Distributed Cloud Console

## ขั้นตอนที่ 1: ไปที่การจัดการบทบาท

1. เข้าสู่ระบบ F5 Distributed Cloud Console
2. คลิกที่ไทล์ **Administration** ในหน้าแรก
3. ในเมนูนำทางด้านซ้าย เลือก **IAM** &gt; **Roles**
4. คลิก **+ Add Role**

## ขั้นตอนที่ 2: ตั้งชื่อบทบาท

ป้อน `example-cert-admin` ในช่อง **Role Name**

กฎการตั้งชื่อบทบาท:

- ใช้ตัวอักษรพิมพ์เล็ก ตัวเลข และเครื่องหมายขีดกลางเท่านั้น
- ต้องเริ่มต้นด้วยตัวอักษรพิมพ์เล็กสองตัว
- ความยาวสูงสุด 64 ตัวอักษร

## ขั้นตอนที่ 3: เพิ่มกลุ่ม Proxy API

สิทธิ์เกี่ยวกับใบรับรองเป็นส่วนหนึ่งของพื้นที่ฟังก์ชัน proxy เพิ่ม
กลุ่ม API ต่อไปนี้เพื่อให้สิทธิ์การจัดการใบรับรอง:

1. คลิก **+ Allowed API Groups**
2. ในช่องค้นหา/กรอง พิมพ์ `ves-io-proxy`
3. เลือกกลุ่ม API ต่อไปนี้:

| กลุ่ม API | สิทธิ์ที่ให้ |
| --------- | ------ |
| `ves-io-proxy-read` | สิทธิ์อ่านทรัพยากร proxy รวมถึงใบรับรอง |
| `ves-io-proxy-write` | สิทธิ์เขียนทรัพยากร proxy รวมถึงใบรับรอง |

4. คลิก **Save** เพื่อยืนยันกลุ่ม API ที่เลือก

:::note
กลุ่มเหล่านี้ครอบคลุมทรัพยากรที่เกี่ยวข้องกับ proxy ทั้งหมด (HTTP Load Balancers, origins,
routes, ใบรับรอง, สายใบรับรอง ฯลฯ) ไม่ใช่เฉพาะใบรับรองเท่านั้น F5 XC
จัดระเบียบสิทธิ์ตามพื้นที่ฟังก์ชัน ไม่ใช่ตามประเภททรัพยากรแต่ละรายการ
:::

## ขั้นตอนที่ 4: บันทึกบทบาท

ตรวจสอบกลุ่ม API ที่เลือกในสรุป จากนั้นคลิก **Save** เพื่อสร้าง
บทบาท

## ขั้นตอนที่ 5: ตรวจสอบบทบาท

1. ไปที่ **IAM** &gt; **Roles**
2. คลิก **example-cert-admin** ในรายการบทบาท
3. ยืนยันว่ากลุ่ม API ทั้งสองรายการแสดงอยู่

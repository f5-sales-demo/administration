คู่มือนี้จะแนะนำขั้นตอนการสร้างบทบาท RBAC ที่กำหนดเองโดยใช้ F5 Distributed
Cloud Console UI บทบาทนี้ให้สิทธิ์ในการจัดการใบรับรอง SSL/TLS และ
ห่วงโซ่ใบรับรองที่ใช้โดย HTTP Load Balancers สำหรับแนวทางแบบโปรแกรม (API)
ดูที่ [คู่มือ API](./custom-role-api)

## หลักการทำงานของ F5 XC RBAC

F5 XC ใช้โมเดลสิทธิ์แบบสามระดับ:

```
api_group_element  (read-only, system-defined)
       │
       ▼
   api_group       (read-only, system-defined)
       │
       ▼
     role           (full CRUD — this is what you create)
```

1. **`api_group_element`** — กำหนดเส้นทาง regex และ HTTP methods
   (เช่น `POST /api/config/.*/certificates`) จัดการโดยระบบ คุณไม่สามารถ
   สร้าง แก้ไข หรือลบสิ่งเหล่านี้ได้

2. **`api_group`** — กลุ่มที่มีชื่อของการอ้างอิง `api_group_element`
   จัดระเบียบตามพื้นที่การทำงาน จัดการโดยระบบ คุณไม่สามารถสร้าง แก้ไข หรือ
   ลบสิ่งเหล่านี้ได้ กลุ่มใช้รูปแบบการตั้งชื่อเช่น `ves-io-proxy-read` หรือ
   `f5xc-waap-standard-admin`

3. **`role`** — อ้างอิงชื่อ `api_group` ในอาร์เรย์ **นี่คือระดับเดียว
   ที่คุณสร้างได้** ใช้ endpoint ของบทบาทที่กำหนดเองเพื่อแนบ
   `api_groups` เฉพาะกับบทบาท

การเลือกกลุ่ม proxy read/write ทำให้บทบาท `example-cert-admin` ให้สิทธิ์การจัดการใบรับรอง SSL/TLS
เป็นส่วนหนึ่งของการเข้าถึงทรัพยากร proxy ที่กว้างขึ้น

## ขอบเขตและข้อจำกัด

การเลือกกลุ่ม proxy read/write จะให้สิทธิ์เข้าถึง **ทรัพยากร proxy ทั้งหมด**
ใน namespace ที่กำหนด — HTTP Load Balancers, origins, routes และ
certificates ทั้งหมด ไม่มีวิธีจำกัดบทบาทให้เฉพาะ certificates เท่านั้น

F5 XC ไม่รองรับการสร้าง `api_group` หรือ `api_group_element`
โดยผู้ใช้ แพลตฟอร์มกำหนดสิ่งเหล่านี้ไว้ล่วงหน้าเป็นแบบอ่านอย่างเดียว และบทบาทสามารถ
อ้างอิงเฉพาะชื่อกลุ่มที่มีอยู่แล้วเท่านั้น ไม่มีกลุ่มที่กำหนดโดยระบบที่จำกัดขอบเขต
เฉพาะการดำเนินการใบรับรองเท่านั้น

**การแก้ไข — การแยก namespace:** วางทรัพยากรใบรับรองใน
namespace เฉพาะและกำหนดขอบเขตบทบาท `example-cert-admin` ให้กับ namespace นั้น สิ่งนี้
ป้องกันไม่ให้ผู้ใช้เข้าถึงทรัพยากร proxy ใน namespace อื่น ในขณะที่
ยังคงให้สิทธิ์การจัดการใบรับรองอย่างเต็มรูปแบบภายใน namespace ที่แยกไว้

## ข้อกำหนดเบื้องต้น

- สิทธิ์ Admin ในการเข้าถึง F5 Distributed Cloud Console

## ขั้นตอนที่ 1: นำทางไปยังการจัดการบทบาท

1. เข้าสู่ระบบ F5 Distributed Cloud Console
2. คลิกไทล์ **Administration** ที่หน้าหลัก
3. ในเมนูนำทางด้านซ้าย เลือก **IAM** &gt; **Roles**
4. คลิก **+ Add Role**

## ขั้นตอนที่ 2: ตั้งชื่อบทบาท

ป้อน `example-cert-admin` ในฟิลด์ **Role Name**

กฎการตั้งชื่อบทบาท:

- ใช้ได้เฉพาะตัวอักษรพิมพ์เล็ก ตัวเลข และเครื่องหมายยัติภังค์
- ต้องขึ้นต้นด้วยตัวอักษรพิมพ์เล็กสองตัว
- ความยาวสูงสุด 64 ตัวอักษร

## ขั้นตอนที่ 3: เพิ่มกลุ่ม API ของ Proxy

สิทธิ์ใบรับรองเป็นส่วนหนึ่งของพื้นที่การทำงาน proxy เพิ่ม
กลุ่ม API ต่อไปนี้เพื่อให้สิทธิ์การจัดการใบรับรอง:

1. คลิก **+ Allowed API Groups**
2. ในฟิลด์ค้นหา/กรอง พิมพ์ `ves-io-proxy`
3. เลือกกลุ่ม API ต่อไปนี้:

| กลุ่ม API | สิทธิ์ที่ได้รับ |
| --------- | ------ |
| `ves-io-proxy-read` | สิทธิ์อ่านทรัพยากร proxy รวมถึงใบรับรอง |
| `ves-io-proxy-write` | สิทธิ์เขียนทรัพยากร proxy รวมถึงใบรับรอง |

4. คลิก **Save** เพื่อยืนยันกลุ่ม API ที่เลือก

:::note
กลุ่มเหล่านี้ครอบคลุมทรัพยากรที่เกี่ยวข้องกับ proxy ทั้งหมด (HTTP Load Balancers, origins,
routes, certificates, certificate chains เป็นต้น) ไม่ใช่เฉพาะ certificates เท่านั้น F5 XC
จัดระเบียบสิทธิ์ตามพื้นที่การทำงาน ไม่ใช่ตามประเภททรัพยากรเดี่ยว
:::

## ขั้นตอนที่ 4: บันทึกบทบาท

ตรวจสอบกลุ่ม API ที่เลือกในส่วนสรุป จากนั้นคลิก **Save** เพื่อสร้าง
บทบาท

## ขั้นตอนที่ 5: ตรวจสอบบทบาท

1. นำทางไปยัง **IAM** &gt; **Roles**
2. คลิก **example-cert-admin** ในรายการบทบาท
3. ยืนยันว่ากลุ่ม API ทั้งสองแสดงอยู่ในรายการ

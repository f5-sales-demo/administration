يرشدك هذا الدليل خطوة بخطوة لإنشاء دور RBAC مخصص باستخدام واجهة مستخدم وحدة تحكم F5 Distributed Cloud. يمنح هذا الدور صلاحية إدارة شهادات SSL/TLS وسلاسل الشهادات المستخدمة بواسطة موازنات أحمال HTTP. للاطلاع على النهج البرمجي (API)، راجع [دليل API](./custom-role-api).

## كيف يعمل نظام RBAC في F5 XC

يستخدم F5 XC نموذج أذونات من ثلاث مستويات:

```
api_group_element  (read-only, system-defined)
       │
       ▼
   api_group       (read-only, system-defined)
       │
       ▼
     role           (full CRUD — this is what you create)
```

1. **`api_group_element`** — يحدد مسار تعبير نمطي (regex) وطرق HTTP
   (مثل `POST /api/config/.*/certificates`). يُدار بواسطة النظام؛ ولا يمكنك
   إنشاؤها أو تعديلها أو حذفها.

2. **`api_group`** — مجموعة مسماة من مراجع `api_group_element`،
   منظمة حسب المجال الوظيفي. يُدار بواسطة النظام؛ ولا يمكنك إنشاؤها أو تعديلها أو
   حذفها. تستخدم المجموعات اصطلاحات تسمية مثل `ves-io-proxy-read` أو
   `f5xc-waap-standard-admin`.

3. **`role`** — يشير إلى أسماء `api_group` في مصفوفة. **هذا هو المستوى الوحيد
   الذي تقوم بإنشائه.** استخدم نقاط نهاية الدور المخصص لربط مجموعات
   `api_groups` محددة بدور معين.

من خلال اختيار مجموعات القراءة/الكتابة للبروكسي، يمنح دور `example-cert-admin` صلاحية إدارة شهادات SSL/TLS كجزء من الوصول الأوسع لموارد البروكسي.

## النطاق والقيود

يمنح اختيار مجموعات القراءة/الكتابة للبروكسي الوصول إلى **جميع موارد البروكسي**
في مساحة الاسم المعيّنة — موازنات أحمال HTTP، والأصول (origins)، والمسارات (routes)، والشهادات
على حد سواء. لا توجد طريقة لتقييد الدور بالشهادات فقط.

لا يدعم F5 XC إنشاء كائنات `api_group` أو `api_group_element` من قِبل المستخدم.
تُحدد المنصة هذه الكائنات مسبقاً كعناصر للقراءة فقط، ولا يمكن للأدوار سوى
الإشارة إلى أسماء المجموعات الموجودة. لا توجد مجموعة معرّفة مسبقاً في النظام مقتصرة حصرياً
على عمليات الشهادات.

**التخفيف — عزل مساحة الاسم:** ضع موارد الشهادات في
مساحة اسم مخصصة وحدد نطاق دور `example-cert-admin` لتلك المساحة. هذا
يمنع المستخدم من الوصول إلى موارد البروكسي في مساحات الاسم الأخرى مع
الاستمرار في منح إدارة كاملة للشهادات داخل مساحة الاسم المعزولة.

## المتطلبات الأساسية

- صلاحية وصول المسؤول إلى وحدة تحكم F5 Distributed Cloud

## الخطوة 1: الانتقال إلى إدارة الأدوار

1. سجّل الدخول إلى وحدة تحكم F5 Distributed Cloud
2. انقر على لوحة **Administration** في الصفحة الرئيسية
3. في قائمة التنقل الجانبية، اختر **IAM** &gt; **Roles**
4. انقر على **+ Add Role**

## الخطوة 2: تسمية الدور

أدخل `example-cert-admin` في حقل **Role Name**.

قواعد تسمية الدور:

- أحرف صغيرة وأرقام وشرطات فقط
- يجب أن يبدأ بحرفين صغيرين
- بحد أقصى 64 حرفاً

## الخطوة 3: إضافة مجموعات API للبروكسي

أذونات الشهادات هي جزء من المجال الوظيفي للبروكسي. أضف
مجموعات API التالية لمنح صلاحية إدارة الشهادات:

1. انقر على **+ Allowed API Groups**
2. في حقل البحث/التصفية، اكتب `ves-io-proxy`
3. اختر مجموعات API التالية:

| مجموعة API | الصلاحيات الممنوحة |
| --------- | ------ |
| `ves-io-proxy-read` | صلاحية قراءة موارد البروكسي بما في ذلك الشهادات |
| `ves-io-proxy-write` | صلاحية كتابة موارد البروكسي بما في ذلك الشهادات |

4. انقر على **Save** لتأكيد مجموعات API المحددة

:::note
تغطي هذه المجموعات جميع الموارد المتعلقة بالبروكسي (موازنات أحمال HTTP، والأصول،
والمسارات، والشهادات، وسلاسل الشهادات، وغيرها)، وليس الشهادات فقط. ينظّم F5 XC
الأذونات حسب المجال الوظيفي، وليس حسب نوع المورد الفردي.
:::

## الخطوة 4: حفظ الدور

راجع مجموعات API المحددة في الملخص، ثم انقر على **Save** لإنشاء
الدور.

## الخطوة 5: التحقق من الدور

1. انتقل إلى **IAM** &gt; **Roles**
2. انقر على **example-cert-admin** في قائمة الأدوار
3. تأكد من إدراج مجموعتي API

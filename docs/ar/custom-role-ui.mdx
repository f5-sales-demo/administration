يرشدك هذا الدليل خلال إنشاء دور RBAC مخصص باستخدام واجهة مستخدم وحدة تحكم F5 Distributed
Cloud. يمنح الدور صلاحية إدارة شهادات SSL/TLS وسلاسل الشهادات المستخدمة بواسطة موازنات أحمال HTTP. للاطلاع على الطريقة البرمجية (API)، راجع [دليل API](./custom-role-api).

## كيف يعمل RBAC في F5 XC

يستخدم F5 XC نموذج أذونات من ثلاثة مستويات:

```
api_group_element  (read-only, system-defined)
       │
       ▼
   api_group       (read-only, system-defined)
       │
       ▼
     role           (full CRUD — this is what you create)
```

1. **`api_group_element`** — يحدد مسار regex وطرق HTTP
   (مثل `POST /api/config/.*/certificates`). يُدار بواسطة النظام؛ لا يمكنك
   إنشاؤه أو تعديله أو حذفه.

2. **`api_group`** — مجموعة مسماة من مراجع `api_group_element`،
   منظمة حسب المجال الوظيفي. يُدار بواسطة النظام؛ لا يمكنك إنشاؤه أو تعديله أو
   حذفه. تستخدم المجموعات اصطلاحات تسمية مثل `ves-io-proxy-read` أو
   `f5xc-waap-standard-admin`.

3. **`role`** — يشير إلى أسماء `api_group` في مصفوفة. **هذا هو المستوى الوحيد
   الذي تقوم بإنشائه.** استخدم نقاط نهاية الأدوار المخصصة لربط
   `api_groups` محددة بدور.

من خلال اختيار مجموعات القراءة/الكتابة الخاصة بالبروكسي، يمنح دور `example-cert-admin` إدارة شهادات SSL/TLS
كجزء من الوصول الأوسع لموارد البروكسي.

## النطاق والقيود

يمنح اختيار مجموعات القراءة/الكتابة الخاصة بالبروكسي الوصول إلى **جميع موارد البروكسي**
في نطاق الأسماء المعيّن — موازنات أحمال HTTP، والمصادر، والمسارات، والشهادات
على حد سواء. لا توجد طريقة لتقييد الدور بالشهادات فقط.

لا يدعم F5 XC إنشاء كائنات `api_group` أو `api_group_element`
بواسطة المستخدم. تحدد المنصة هذه الكائنات مسبقاً للقراءة فقط، ولا يمكن للأدوار سوى
الإشارة إلى أسماء المجموعات الموجودة. لا توجد مجموعة محددة من النظام مخصصة حصرياً
لعمليات الشهادات.

**التخفيف — عزل نطاق الأسماء:** ضع موارد الشهادات في
نطاق أسماء مخصص وحدد نطاق دور `example-cert-admin` لذلك النطاق. هذا
يمنع المستخدم من الوصول إلى موارد البروكسي في نطاقات أسماء أخرى مع
الاستمرار في منح إدارة كاملة للشهادات ضمن نطاق الأسماء المعزول.

## المتطلبات الأساسية

- صلاحية المسؤول للوصول إلى وحدة تحكم F5 Distributed Cloud

## الخطوة 1: الانتقال إلى إدارة الأدوار

1. سجل الدخول إلى وحدة تحكم F5 Distributed Cloud
2. انقر على لوحة **Administration** في الصفحة الرئيسية
3. في قائمة التنقل اليسرى، اختر **IAM** &gt; **Roles**
4. انقر على **+ Add Role**

## الخطوة 2: تسمية الدور

أدخل `example-cert-admin` في حقل **Role Name**.

قواعد تسمية الدور:

- أحرف صغيرة وأرقام وواصلات فقط
- يجب أن يبدأ بحرفين صغيرين
- بحد أقصى 64 حرفاً

## الخطوة 3: إضافة مجموعات API الخاصة بالبروكسي

أذونات الشهادات هي جزء من المجال الوظيفي للبروكسي. أضف
مجموعات API التالية لمنح صلاحية إدارة الشهادات:

1. انقر على **+ Allowed API Groups**
2. في حقل البحث/التصفية، اكتب `ves-io-proxy`
3. اختر مجموعات API التالية:

| مجموعة API | الصلاحيات الممنوحة |
| --------- | ------ |
| `ves-io-proxy-read` | صلاحية القراءة لموارد البروكسي بما في ذلك الشهادات |
| `ves-io-proxy-write` | صلاحية الكتابة لموارد البروكسي بما في ذلك الشهادات |

4. انقر على **Save** لتأكيد مجموعات API المحددة

:::note
تغطي هذه المجموعات جميع الموارد المتعلقة بالبروكسي (موازنات أحمال HTTP، والمصادر،
والمسارات، والشهادات، وسلاسل الشهادات، إلخ)، وليس الشهادات فقط. ينظم F5 XC
الأذونات حسب المجال الوظيفي، وليس حسب نوع المورد الفردي.
:::

## الخطوة 4: حفظ الدور

راجع مجموعات API المحددة في الملخص، ثم انقر على **Save** لإنشاء
الدور.

## الخطوة 5: التحقق من الدور

1. انتقل إلى **IAM** &gt; **Roles**
2. انقر على **example-cert-admin** في قائمة الأدوار
3. تأكد من إدراج كلتا مجموعتي API

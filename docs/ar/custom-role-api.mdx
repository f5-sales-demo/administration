يرشدك هذا الدليل خلال إنشاء دور RBAC مخصص في F5 Distributed Cloud
يمنح صلاحية إدارة شهادات SSL/TLS وسلاسل الشهادات
المستخدمة بواسطة HTTP Load Balancers. للاطلاع على النهج المعتمد على واجهة المستخدم، راجع
[دليل وحدة التحكم المبسط](./custom-role-ui).

## كيف يعمل RBAC في F5 XC

يستخدم F5 XC نموذج أذونات من ثلاث مستويات:

```
api_group_element  (للقراءة فقط، محدد من النظام)
       │
       ▼
   api_group       (للقراءة فقط، محدد من النظام)
       │
       ▼
     role           (CRUD كامل — هذا ما تقوم بإنشائه)
```

1. **`api_group_element`** — يحدد مسار regex وطرق HTTP
   (مثل `POST /api/config/.*/certificates`). يُدار من النظام؛ لا يمكنك
   إنشاؤها أو تعديلها أو حذفها.

2. **`api_group`** — مجموعة مسماة من مراجع `api_group_element`،
   منظمة حسب المنطقة الوظيفية. تُدار من النظام؛ لا يمكنك إنشاؤها أو تعديلها أو
   حذفها. تستخدم المجموعات اصطلاحات تسمية مثل `ves-io-proxy-read` أو
   `f5xc-waap-standard-admin`.

3. **`role`** — يشير إلى أسماء `api_group` في مصفوفة. **هذا هو المستوى الوحيد
   الذي تقوم بإنشائه.** استخدم نقاط نهاية الدور المخصص لربط
   `api_groups` محددة بدور ما.

عمليات الشهادات لـ HTTP Load Balancers مُجمّعة في api_groups الخاصة بـ
`ves-io-proxy-*` إلى جانب موارد البروكسي الأخرى (HTTP Load
Balancers، المسارات، الأصول، إلخ). لا توجد api_groups مخصصة للشهادات فقط — تنظم المنصة الأذونات حسب المنطقة الوظيفية، وليس حسب
نوع المورد الفردي.

## النطاق والقيود

تمنح api_groups الخاصة بـ `ves-io-proxy-read` و `ves-io-proxy-write` الوصول إلى
**جميع الموارد المتعلقة بالبروكسي**، وليس الشهادات فقط. يمكن للمستخدم المعيّن لهذا
الدور أيضاً إدارة HTTP Load Balancers والأصول والمسارات وكائنات
نطاق البروكسي الأخرى.

RBAC المقتصر على الشهادات فقط غير ممكن في F5 XC. كل من `api_group` و
`api_group_element` كائنات محددة من النظام وللقراءة فقط — تكشف واجهة برمجة التطبيقات فقط عن نقاط نهاية GET لها. يمكن للأدوار فقط الإشارة إلى أسماء `api_group`
الموجودة، ولا يوجد `api_group` مخصص حصرياً للشهادات.

**التخفيف — عزل مساحة الأسماء:** أنشئ مساحة أسماء مخصصة لموارد
الشهادات وعيّن دور `example-cert-admin` محدود النطاق بتلك
المساحة فقط. يحتفظ المستخدم بأذونات مستوى البروكسي لكن لا يمكنه الوصول إلى
الموارد في مساحات الأسماء الأخرى، مما يحد من نطاق التأثير لنطاق
api_group الأوسع.

## المتطلبات الأساسية

- رمز API لـ F5 XC بصلاحيات المسؤول
- `curl` و `jq` مثبتان

عيّن متغيرات البيئة الخاصة بك:

```bash
export F5XC_TENANT="your-tenant"
export F5XC_API_URL="https://${F5XC_TENANT}.console.ves.volterra.io"
export F5XC_API_TOKEN="your-api-token"
```

## الخطوة 1: اكتشاف مجموعات API الخاصة بالبروكسي

اعرض جميع كائنات `api_group` المحددة من النظام وفلترها للإدخالات المتعلقة بالبروكسي
(التي تتضمن أذونات الشهادات):

```bash
curl -s \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  "${F5XC_API_URL}/api/web/namespaces/shared/api_groups" \
  | jq '[.items[] | select(.name | test("ves-io-proxy")) | .name] | sort'
```

المخرجات المتوقعة:

```json
[
  "ves-io-proxy-app-firewall-read",
  "ves-io-proxy-app-firewall-write",
  "ves-io-proxy-monitor-read",
  "ves-io-proxy-monitor-write",
  "ves-io-proxy-read",
  "ves-io-proxy-security-read",
  "ves-io-proxy-security-write",
  "ves-io-proxy-waf-read",
  "ves-io-proxy-waf-write",
  "ves-io-proxy-write"
]
```

لإدارة الشهادات، المجموعات ذات الصلة هي:

| مجموعة API | تمنح |
| --------- | ------ |
| `ves-io-proxy-read` | وصول القراءة لموارد البروكسي بما في ذلك الشهادات |
| `ves-io-proxy-write` | وصول الكتابة لموارد البروكسي بما في ذلك الشهادات |

:::note
تغطي هذه المجموعات جميع الموارد المتعلقة بالبروكسي (HTTP Load Balancers، الأصول،
المسارات، الشهادات، سلاسل الشهادات، إلخ)، وليس الشهادات فقط. ينظم F5 XC
الأذونات حسب المنطقة الوظيفية. إذا كنت بحاجة إلى التحقق من مسارات API المحددة
التي تمنحها مجموعة ما، تحقق من نقطة نهاية `api_group_elements` الموصوفة في
قسم مرجع API.
:::

## الخطوة 2: إنشاء الدور المخصص

استخدم نقطة نهاية إنشاء الدور المخصص. تشير مصفوفة `api_groups` إلى
أسماء api_group المحددة من النظام المكتشفة في الخطوة 1.

أنشئ الدور المخصص بوصول القراءة والكتابة لموارد البروكسي (بما في ذلك
الشهادات وسلاسل الشهادات):

```bash
curl -s -X POST \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  -H "Content-Type: application/json" \
  "${F5XC_API_URL}/api/web/custom/namespaces/system/roles" \
  -d '{
    "metadata": {
      "name": "example-cert-admin",
      "namespace": "system"
    },
    "spec": {},
    "api_groups": [
      "ves-io-proxy-read",
      "ves-io-proxy-write"
    ]
  }'
```

## الخطوة 3: التحقق من الدور

استرجع الدور باستخدام نقطة نهاية GET المخصصة للتأكد من أن `api_groups`
مرتبطة:

```bash
curl -s \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  "${F5XC_API_URL}/api/web/custom/namespaces/system/roles/example-cert-admin" \
  | jq '{name: .object.metadata.name, api_groups: .api_groups}'
```

المخرجات المتوقعة:

```json
{
  "name": "example-cert-admin",
  "api_groups": [
    "ves-io-proxy-read",
    "ves-io-proxy-write"
  ]
}
```

## الخطوة 4: تحديث الدور (اختياري)

لإضافة أو إزالة `api_groups` من دور موجود، استخدم نقطة نهاية PUT
المخصصة:

```bash
curl -s -X PUT \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  -H "Content-Type: application/json" \
  "${F5XC_API_URL}/api/web/custom/namespaces/system/roles/example-cert-admin" \
  -d '{
    "name": "example-cert-admin",
    "namespace": "system",
    "spec": {},
    "api_groups": [
      "ves-io-proxy-read"
    ]
  }'
```

يقوم هذا المثال بتخفيض مستوى الدور إلى وصول القراءة فقط عن طريق إزالة
`ves-io-proxy-write`.

## الخطوة 5: حذف الدور (اختياري)

يستخدم الحذف نقطة نهاية الدور القياسية:

```bash
curl -s -X DELETE \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  "${F5XC_API_URL}/api/web/namespaces/system/roles/example-cert-admin"
```

## مرجع API

### نقاط نهاية الأدوار (مخصصة — تتضمن `api_groups`)

الأدوار دائماً في مساحة أسماء `system`.

| الطريقة | المسار | الوصف |
| ------ | ---- | ----------- |
| POST | `/api/web/custom/namespaces/system/roles` | إنشاء دور مخصص |
| GET | `/api/web/custom/namespaces/system/roles` | عرض الأدوار المخصصة |
| GET | `/api/web/custom/namespaces/system/roles/{name}` | الحصول على دور مخصص |
| PUT | `/api/web/custom/namespaces/system/roles/{name}` | استبدال دور مخصص |

### نقاط نهاية الأدوار (قياسية)

| الطريقة | المسار | الوصف |
| ------ | ---- | ----------- |
| DELETE | `/api/web/namespaces/system/roles/{name}` | حذف دور |
| GET | `/api/web/namespaces/system/roles/{name}` | الحصول على دور |
| GET | `/api/web/namespaces/system/roles` | عرض الأدوار |

### نقاط نهاية الاكتشاف (للقراءة فقط)

يستخدم الاكتشاف مساحة أسماء `shared`.

| الطريقة | المسار | الوصف |
| ------ | ---- | ----------- |
| GET | `/api/web/namespaces/shared/api_groups` | عرض جميع api_groups |
| GET | `/api/web/namespaces/shared/api_group_elements` | عرض جميع api_group_elements |

### نقاط نهاية الشهادات (ما يمنح الدور الوصول إليه)

استبدل `{namespace}` بمساحة الأسماء حيث تتم إدارة شهاداتك.

| الطريقة | المسار | الوصف |
| ------ | ---- | ----------- |
| POST | `/api/config/namespaces/{namespace}/certificates` | إنشاء شهادة |
| GET | `/api/config/namespaces/{namespace}/certificates` | عرض الشهادات |
| GET | `/api/config/namespaces/{namespace}/certificates/{name}` | الحصول على شهادة |
| PUT | `/api/config/namespaces/{namespace}/certificates/{name}` | استبدال شهادة |
| DELETE | `/api/config/namespaces/{namespace}/certificates/{name}` | حذف شهادة |
| POST | `/api/config/namespaces/{namespace}/certificate_chains` | إنشاء سلسلة شهادات |
| GET | `/api/config/namespaces/{namespace}/certificate_chains` | عرض سلاسل الشهادات |
| GET | `/api/config/namespaces/{namespace}/certificate_chains/{name}` | الحصول على سلسلة شهادات |
| PUT | `/api/config/namespaces/{namespace}/certificate_chains/{name}` | استبدال سلسلة شهادات |
| DELETE | `/api/config/namespaces/{namespace}/certificate_chains/{name}` | حذف سلسلة شهادات |

## لماذا `api_group` و `api_group_element` للقراءة فقط

هذه الكائنات **محددة من النظام** بواسطة F5 XC. تقوم المنصة بإنشاء
إدخالات `api_group_element` مسبقاً لكل مسار API وتجميعها في
كائنات `api_group` منظمة حسب المنطقة الوظيفية.

لا تحتاج إلى إنشاء هذه — فهي موجودة بالفعل لكل مورد في
النظام. مهمتك هي **اكتشاف** أسماء المجموعات ذات الصلة (الخطوة 1) و**الإشارة** إليها في دورك المخصص (الخطوة 2).

لمعرفة مسارات API الفردية التي تغطيها api_group، اعرض
نقطة نهاية `api_group_elements` وابحث عن العناصر المتعلقة بموردك:

```bash
curl -s \
  -H "Authorization: APIToken ${F5XC_API_TOKEN}" \
  "${F5XC_API_URL}/api/web/namespaces/shared/api_group_elements" \
  | jq '[.items[] | select(.name | test("ves-io-schema-certificate")) | .name]'
```

このガイドでは、F5 Distributed Cloud Console UIを使用してカスタムRBACロールを作成する手順を説明します。このロールは、HTTP Load Balancerで使用されるSSL/TLS証明書および証明書チェーンの管理アクセスを付与します。プログラム（API）によるアプローチについては、[APIガイド](./custom-role-api)を参照してください。

## F5 XC RBACの仕組み

F5 XCは3層の権限モデルを使用しています：

```
api_group_element  (読み取り専用、システム定義)
       │
       ▼
   api_group       (読み取り専用、システム定義)
       │
       ▼
     role           (完全なCRUD — これがユーザーが作成するもの)
```

1. **`api_group_element`** — 正規表現パスとHTTPメソッドを定義します（例：`POST /api/config/.*/certificates`）。システム管理であり、作成、変更、削除はできません。

2. **`api_group`** — `api_group_element`参照の名前付きコレクションで、機能領域ごとに整理されています。システム管理であり、作成、変更、削除はできません。グループは`ves-io-proxy-read`や`f5xc-waap-standard-admin`のような命名規則を使用します。

3. **`role`** — `api_group`名を配列で参照します。**これがユーザーが作成する唯一の層です。**カスタムロールエンドポイントを使用して、特定の`api_groups`をロールに紐付けます。

プロキシの読み取り/書き込みグループを選択することで、`example-cert-admin`ロールは、より広範なプロキシリソースアクセスの一部としてSSL/TLS証明書管理を付与します。

## スコープと制限事項

プロキシの読み取り/書き込みグループを選択すると、割り当てられた名前空間内の**すべてのプロキシリソース**へのアクセスが付与されます — HTTP Load Balancer、オリジン、ルート、および証明書のすべてが対象となります。ロールを証明書のみに制限する方法はありません。

F5 XCは、ユーザーが作成する`api_group`または`api_group_element`オブジェクトをサポートしていません。プラットフォームはこれらを読み取り専用として事前定義しており、ロールは既存のグループ名のみを参照できます。証明書操作のみにスコープされたシステム定義のグループは存在しません。

**緩和策 — 名前空間の分離：** 証明書リソースを専用の名前空間に配置し、`example-cert-admin`ロールのスコープをその名前空間に限定します。これにより、ユーザーが他の名前空間のプロキシリソースにアクセスすることを防ぎつつ、分離された名前空間内での完全な証明書管理を付与できます。

## 前提条件

- F5 Distributed Cloud Consoleへの管理者アクセス

## ステップ1：ロール管理への移動

1. F5 Distributed Cloud Consoleにログインします
2. ホームページの**Administration**タイルをクリックします
3. 左側のナビゲーションメニューで、**IAM** &gt; **Roles**を選択します
4. **+ Add Role**をクリックします

## ステップ2：ロールの命名

**Role Name**フィールドに`example-cert-admin`と入力します。

ロールの命名規則：

- 小文字、数字、ハイフンのみ使用可能
- 2文字の小文字で始める必要があります
- 最大64文字

## ステップ3：プロキシAPIグループの追加

証明書の権限はプロキシ機能領域の一部です。証明書管理アクセスを付与するために、以下のAPIグループを追加します：

1. **+ Allowed API Groups**をクリックします
2. 検索/フィルターフィールドに`ves-io-proxy`と入力します
3. 以下のAPIグループを選択します：

| APIグループ | 付与される権限 |
| --------- | ------ |
| `ves-io-proxy-read` | 証明書を含むプロキシリソースへの読み取りアクセス |
| `ves-io-proxy-write` | 証明書を含むプロキシリソースへの書き込みアクセス |

4. **Save**をクリックして選択したAPIグループを確定します

:::note
これらのグループは、証明書だけでなく、すべてのプロキシ関連リソース（HTTP Load Balancer、オリジン、ルート、証明書、証明書チェーンなど）をカバーします。F5 XCは個別のリソースタイプではなく、機能領域ごとに権限を整理しています。
:::

## ステップ4：ロールの保存

サマリーで選択したAPIグループを確認し、**Save**をクリックしてロールを作成します。

## ステップ5：ロールの確認

1. **IAM** &gt; **Roles**に移動します
2. ロール一覧で**example-cert-admin**をクリックします
3. 両方のAPIグループが表示されていることを確認します

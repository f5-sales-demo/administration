本指南將逐步說明如何使用 F5 Distributed Cloud 主控台 UI 建立自訂 RBAC 角色。此角色授予管理 HTTP 負載平衡器所使用的 SSL/TLS 憑證和憑證鏈的存取權限。如需程式化（API）方式，請參閱 [API 指南](./custom-role-api)。

## F5 XC RBAC 的運作方式

F5 XC 使用三層權限模型：

```
api_group_element  (唯讀，系統定義)
       │
       ▼
   api_group       (唯讀，系統定義)
       │
       ▼
     role           (完整 CRUD — 這是您所建立的)
```

1. **`api_group_element`** — 定義正規表達式路徑和 HTTP 方法（例如 `POST /api/config/.*/certificates`）。由系統管理；您無法建立、修改或刪除這些項目。

2. **`api_group`** — `api_group_element` 參考的具名集合，依功能領域組織。由系統管理；您無法建立、修改或刪除這些項目。群組使用如 `ves-io-proxy-read` 或 `f5xc-waap-standard-admin` 的命名慣例。

3. **`role`** — 在陣列中參考 `api_group` 名稱。**這是您唯一需要建立的層級。** 使用自訂角色端點將特定的 `api_groups` 附加到角色。

透過選擇 proxy 讀取/寫入群組，`example-cert-admin` 角色會授予 SSL/TLS 憑證管理權限，作為更廣泛的 proxy 資源存取的一部分。

## 範圍與限制

選擇 proxy 讀取/寫入群組會授予對已指派命名空間中**所有 proxy 資源**的存取權限 — 包括 HTTP 負載平衡器、來源、路由和憑證。無法將角色限制為僅存取憑證。

F5 XC 不支援使用者建立 `api_group` 或 `api_group_element` 物件。平台將這些預先定義為唯讀，角色只能參考現有的群組名稱。沒有任何系統定義的群組僅限於憑證操作的範圍。

**緩解措施 — 命名空間隔離：** 將憑證資源放置在專用的命名空間中，並將 `example-cert-admin` 角色的範圍限定在該命名空間。這可防止使用者存取其他命名空間中的 proxy 資源，同時仍在隔離的命名空間內授予完整的憑證管理權限。

## 先決條件

- 具備 F5 Distributed Cloud 主控台的管理員存取權限

## 步驟 1：導覽至角色管理

1. 登入 F5 Distributed Cloud 主控台
2. 在首頁點擊 **Administration** 磚塊
3. 在左側導覽選單中，選擇 **IAM** &gt; **Roles**
4. 點擊 **+ Add Role**

## 步驟 2：命名角色

在 **Role Name** 欄位中輸入 `example-cert-admin`。

角色命名規則：

- 僅限小寫字母、數字和連字號
- 必須以兩個小寫字母開頭
- 最多 64 個字元

## 步驟 3：新增 Proxy API 群組

憑證權限屬於 proxy 功能領域的一部分。新增以下 API 群組以授予憑證管理存取權限：

1. 點擊 **+ Allowed API Groups**
2. 在搜尋/篩選欄位中輸入 `ves-io-proxy`
3. 選擇以下 API 群組：

| API 群組 | 授予的權限 |
| --------- | ------ |
| `ves-io-proxy-read` | 對 proxy 資源（包括憑證）的讀取存取權限 |
| `ves-io-proxy-write` | 對 proxy 資源（包括憑證）的寫入存取權限 |

4. 點擊 **Save** 以確認所選的 API 群組

:::note
這些群組涵蓋所有 proxy 相關資源（HTTP 負載平衡器、來源、路由、憑證、憑證鏈等），而不僅僅是憑證。F5 XC 依功能領域組織權限，而非依個別資源類型。
:::

## 步驟 4：儲存角色

檢閱摘要中所選的 API 群組，然後點擊 **Save** 以建立角色。

## 步驟 5：驗證角色

1. 導覽至 **IAM** &gt; **Roles**
2. 在角色清單中點擊 **example-cert-admin**
3. 確認兩個 API 群組均已列出
